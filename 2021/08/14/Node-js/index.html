Node.js、npm

Aug 14 2021

因為在練習串接 API 時，發現需要用到 Node.js，因此針對這個部分做學習。

Node.js

Node.js是能夠執行 JavaScript 的一個運行環境，
「瀏覽器」是一個能夠運行 JavaScript 的運行環境，
在 Node.js 出現以前 JavaScript 「只能」在瀏覽器上運行。
所以 Node.js能讓 JavaScript 跑在瀏覽器以外（例如：電腦）。
可以在終端機內輸入 JavaScript 語法並且運行。

npm (Node Package Manager)

npm 是一個 node.js 的套件管理工具。
好處在於：
1.統一管理函式庫（更新、下載、刪除）。
2.可以使用別人寫好的函式庫（程式）。

npm init

如果想要在一個 Project 裡面使用 npm ，需要先使用一個指令 npm init

基本上結束後，可以看到這個資料夾底下，新增了一個 Package.json，

npm install

要新增 selenium 的方法很簡單，先搜尋 npm selenium ，
就可以找到已經被發佈在 NPM 上的 selenium，
而指令也很簡單，npm install selenium-webdriver，
接下來會要等一段時間(NPM 會從官方的資料庫去撈函式庫的資料)，
安裝完可以看到現有的資料夾裡面新增了兩個檔案，
node_modules 和 package-lock.json。
指令除了創建了 node_modules，新增整個 selenium-webdriver 的原始碼，
還附加了許多資料夾，因為 Selenium 本身會需要引入其他函式庫才能使用，
所以會自動幫你都裝好。

npm 建立的檔案與資料夾

node_modules：
每個專案要使用的函式庫，
都直接安裝在專案的資料夾底下，
保持每個環境(資料夾)，永遠都不會有相衝突的機會。


package.json：
用來紀錄專案資訊，所以也包含了函式庫資訊，
別人也可以透過這個檔案去知道使用了哪些函式庫並下載安裝。
之後想把 Project 放到另一個環境時，
只需要複製原始碼，和 package.json，
之後執行 npm install，如果後面不加套件名稱的話，
npm 會當作是你想把 package.json 上現在缺的 Dependency 都裝回來。


package-lock.json：
npm install 會照著 package.json 函式庫資訊進行安裝，
而 package.json 函式庫資訊定義了向後（新）兼容依賴，
若有人在 git hub 上 clone 了專案並運行 npm install ，
可能會得到與原專案不同的函式庫版本，
這時候 package-lock.json 就起了很大的作用，
可以根據這個檔案查看當時開發者所使用的版本去下載函式庫，
才不會造成 bug 產生。
每次 npm install / uninstall /update 都會更新。
